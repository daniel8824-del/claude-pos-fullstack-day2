# 상권 분석 스킬

## 분석 순서
1. 시간대별 매출 추이 (라인 차트)
2. 매장별 매출 (막대 차트)
3. 인기 상품 Top 5 (도넛 차트)

## 색상 규칙
- 매출 = cyan #00b4d8
- 강조 = 빨강 #c92a2a

2SKILL을 작성하면 좋은 점

스킬은 활성화될 때만 메모리에 올라가요. 평소엔 메타데이터만 100토큰 정도 차지하고, 사용자가 "상권 분석" 이라는 말을 꺼낼 때만 본문 5000토큰이 로딩돼요. 토큰비가 거의 0이에요.

  2. Codex가 검수 결과 전달

4QMD — 사내 자료 검색 한 줄

어제 설치한 QMD로 사내 자료를 검색해 볼게요. 988개 사내 문서를 1초 내에 찾아줘요. 외부 API 안 거치니까 토큰비 0원.

  1. 터미널에서 한 줄
    qmd search "상권 분석" -n 5
  2. 결과를 점수 순으로 받기
    [0.92] 최근 분석 자료 / [0.87] 작년 시장조사 / [0.74] 경쟁사 보고서… 점수 0.9 이상이면 전문 읽기.
QMD가 빠른 이유 — 988개 문서를 로컬에 임베딩 미리 저장해 두기 때문이에요. 외부 API를 거치지 않으니 토큰비도 0원, 속도도 1초 내.

1Phase 4 — 훅 설치 (24종 중 5종만)

훅은 24가지가 있는데, 오늘은 가장 유용한 5가지만 같이 보고 설치할게요.

훅 이름역할
carefulrm -rf 같은 위험 명령 자동 경고

replay-learnings세션 시작 시 어제 교훈 자동 회상

3/security-review — 보안 검수

배포하기 전에 보안 점검 한 번. /security-review 한 줄이면 AI가 OWASP Top 10 기준으로 코드를 훑어요. 비밀번호 노출·SQL 인젝션·민감 정보 등.

  1. Claude Code에서 한 줄
    /security-review
  2. 결과 확인
    "이 파일에 API 키가 그대로 박혀 있어요" 같은 경고가 나오면 수정.

4Phase 5 — GitHub Pages 배포 라이브

드디어 마지막. 우리가 만든 POS 분석 결과를 실제 웹에 올려요. 강사가 화면에 띄워놓고 같이 합니다.

  1. GitHub에 새 repo 만들기
    gh repo create my-pos-analysis --public --source=. --push
  2. Pages 활성화
    gh api -X POST /repos/{user}/{repo}/pages -f 'source[branch]=main' -f 'source[path]=/'
  3. ~1-2분 후 라이브 URL 확인
    https://{user}.github.io/my-pos-analysis/ — 본인 분석 결과를 친구에게 공유 가능!
참고로 지금 보고 계신 이 페이지도 똑같은 방법으로 만들어졌어요. 사이트 코드는 GitHub repo 에서 직접 보실 수 있습니다.

아니요. 오늘은 정적 HTML + CSV 파일만 씁니다. 설치할 것은 어제 끝낸 Claude Code 외에 context7 MCP 하나뿐이에요.
비용이 어느 정도 드나요?
오늘 쓰는 도구는 거의 다 무료 또는 강사 환경에서 무상 제공됩니다. Claude Code는 회사 계정 (Pro $20 ~ Max $200), 나머지는 모두 무료.
완주하지 못하면 어떻게 하나요?
이 사이트가 그대로 남아 있어요. 끝나고 댁에서 천천히 따라하셔도 됩니다. 만화 8컷 + 영상 + 단계별 실습이 다 여기에 있어요.
만화 그림체가 좀 어색한데?
현재 만화 8컷은 AI 이미지 생성 모델 (gemini-2.5-flash-image, 일명 나노 바나나)로 만들었는데 한국어 텍스트 렌더링이 약한 한계가 있어요. 추후 Codex CLI의 이미지 생성 도구로 다시 만들어 교체할 예정입니다.
