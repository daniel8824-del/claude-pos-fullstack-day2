1핵심 개념 | HOOK

개념정의
HOOK~/.claude/hooks/ 이벤트 트리거. 도구 사용 전 · 사용 후 · 세션 시작 등 시점에 자동 동작.
주요 훅 5종강사 운영 careful · compact-guard · smart-commit · replay-learnings · learn-rule
/security-review슬래시 명령 국제 보안 표준 OWASP Top 10 (비밀번호 노출·SQL 인젝션·민감 정보 등 10대 점검 항목) 기준 자동 보안 검수.

2자주 쓰는 훅 5종

훅역할
carefulrm -rf 등 위험 명령 자동 차단
compact-guard대화 컨텍스트 50% 도달 시 자동 압축
smart-commitgit commit 직전 메시지 자동 점검
replay-learnings세션 시작 시 과거 교훈 자동 회상
learn-rule"기억해" 키워드 감지 시 자동 저장
  • 위험 명령 자동 차단이 필요할 때 → careful
  • 긴 대화에서 토큰비 폭주를 막고 싶을 때 → compact-guard
  • git 메시지 일관성을 맞추고 싶을 때 → smart-commit
  • 어제 한 작업·교훈을 다음 세션에 자동 회상 → replay-learnings
  • SQL 실행 시 자동 EXPLAIN, 결제 함수 수정 시 자동 보안 점검 같은 도메인 HOOK도 직접 작성 가능

7실습 Phase 5 | GitHub Pages 배포

GitHub Pages = 깃허브의 무료 웹사이트 호스팅 서비스. 정적 HTML/CSS/JS만으로 라이브 사이트를 띄울 수 있습니다.

  1. 신규 repo 생성 + 푸시
    gh repo create my-pos-analysis --public --source=. --push
  2. Pages 활성화
    gh api -X POST /repos/{user}/{repo}/pages \
  -f 'source[branch]=main' -f 'source[path]=/'
  3. 라이브 URL 확인
    https://{user}.github.io/my-pos-analysis/ — 빌드 ~1-2분

본 페이지도 동일 방식 배포 | repo: github.com/daniel8824-del/claude-pos-fullstack-day2

§ AI-Ops · 전환 인사이트

전통 POS → AI-Ops 전환 5단계 로드맵

오늘 배운 도구를 POS 운영·개발 조직에 단계적으로 적용하면 어떤 변화가 가능한지.

단계전통 POS 운영 (As-Is)AI-Ops 적용 (To-Be)
1 매출 보고서 점장별 양식 → 본사 통합 매번 1~2일 CLAUDE.md에 통일 양식 → AI 자동 정리
2 분석 요청 시 SQL 매번 새로 작성 → 평균 2~3일 SKILL로 자주 쓰는 분석 패턴 누적 → 30분 내 처리
3 POS 단말 오류 → 새벽 호출 → 출동 점검 HOOK으로 자동 진단·자동 티켓 발행
4 외부 데이터(상권·날씨·트렌드) 수동 합치기 MCP로 외부 시스템 직통 호출 → AI가 자동 결합
5 배포·보안 점검 매번 수동 /security-review + GitHub Pages 자동화

실행 과제

  1. 데이터 표준화 — 매장별 양식 통합 (CLAUDE.md 1차)
  2. 분석 자동화 — 자주 쓰는 패턴 SKILL.md 누적
  3. 운영 자동화 — HOOK 도입 (위험 명령·실수 자동 차단)
  4. 외부 연계 — MCP로 사내 DB·상권 데이터·티켓 시스템 연결
  5. 배포 파이프라인 — 보안 점검 자동화

고려 사항 (POS 특수성)

  • 결제 정확성 100% — AI 자동화는 보조 영역부터, 결제 본 함수는 사람 승인 후
  • 24/7 영업 환경 — 배포 윈도우 제한, 점진적 롤아웃 필수
  • 매장 네트워크 분산 — 단말 다양성·오프라인 대응 고려
  • PG·카드사 연동 — 외부 의존성 변경 시 다중 검수
  • PCI-DSS 등 보안 규제 — 민감 정보 격리·로그 보존

Claude Code HOOK의 동작 범위 | Claude Code 안에서 동작 (개발자 PC). POS 운영 시스템 자체 자동화는 MCP + 외부 자동화 도구(n8n · Airflow 등) 결합으로 단계적 진입. 오늘 배운 것 = 개발 단계 가속, 운영 시스템 자동화는 별도 트랙.
